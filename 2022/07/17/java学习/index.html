java编译过程

第一步，有一个.java后缀的文件，这文件里保存的是自己写的java代码

第二步，使用JDK的命令javac，javac空格+.java后缀的的文件全名包含后缀.
执行第二部后生成一个.clas后缀的文件，这个文件是字节码文件.

第三步，使用JDK命令java，java+文件名（生成的字节码文件的文件名和原来.java的文件名是一致的）

包（package）
包是组织类的一种方式
使用包的主要目的是保证类的唯一性

  • 比如说，在开发中，2个人写了同名的类就会冲突导致编译不能通过
  • 这时，只要把同名的类放进不同的包里.就可以不冲突的使用.

创建包

1.创建包的时候，包名一定是小写的.
2.包的命名，包的名字一般用的是域名，域名反过来写.比如百度的域名是www.baidu.com,那百度的包就叫，com.baidu.wwww
3.包名要和代码路径匹配.因为你创建包的时候，会根据你的包名自动创建一个路径来存放代码
4.如果一个类没有package语句，该类会被放到默认包中

继承的规则

  1. 使用extends继承父类
  2. java中一个子类只能继承一个父类
  3. 一个父类可以被多个子类继承
  4. 子类会继承父类中所有public的字段和方法
  5. 父类中private的字段和方法，子类是不能使用的.（是不能被子类使用，但还是被子类继承的）
  6. 子类的实例中也包含父类的实例，可以使用super关键字引用.
  7. 子类可以继承父类，新子类可以继承（子类继承父类），可以无限套娃继承
  8. 继承不建议超过三层

继承的优点

  • 可以达到代码的复用效果

问：什么是代码复用？

答：子类可以使用父类的代码（除构造方法外所有的）

继承的意义是：为了代码的复用.如果不继承，那么重复的代码会变得非常多.

接口的规则和注意点

  1. 接口中的方法都是抽象方法，接口中的方法不能被具体实现
  2. JDK1.8后接口中的方法可以被具体实现，（在需要具体实现的方法前用default修饰）
  3. 接口中定义的变量，默认是静态常量（public static final修饰）.方法默认是抽象方法（public abstract修饰）

  5. 接口和类的关系是implement，使用implement继承接口.
  6. 一个类继承（implement）了接口，一定要在这个类中重写接口中的方法
  7. 接口和类发生继承（implement）关系后，就可以发生向上转型

一些基本的打印和从键盘获取字符串和整形就不说了.

主要是讲如何通过输入的整形判断出用户是管理员还是普通用户

都知道adm和普通用户（NormalUser）都继承了User

那么我们就通过if语句俩new对象，1就new管理员，2就new普通用户.

顺便把名字也传进去.传入的过程中，因为有继承关系所以就是：主函数调用login方法 –》login根据获取到的数子–》将名字传入对应的对象中.

对象在调用构造方法，因为是继承（继承的话，子类要帮助父类先完成构造），所以name又被传到User中.

然后从login返回给user接收

选择完毕后就要打印菜单了

但是注意了，在上面这么操作的过程中发生了向上转型.（ruturn的子类，用的是Use(父类进行接收)）

向上转型的效果是：不能调用子类的属性和方法，只能调用父类的属性和方法

那怎么办呢.这里就用到了多态的抽象方法

在User类中定义一个抽象（abstract）方法menu()

抽象方法的效果是：需要在子类中重写父类的抽象方法

然后，我需要选择这些选项中的某一项.

第一种方法：我在main中用scanner获取输入的数字，在调用对应的方法.

第二种方法：我可以直接在menu方法中获取用户输入的数字.通过return直接返回.在主函数中直接接收返回值就好了.

在修改menu()时记得要修改void为int,也别忘了把User中的抽象方法也改成int.
