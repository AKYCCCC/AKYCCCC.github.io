Runtime

Runtime表示当前虚拟机的运行环境

Runtime不是工具类不能直接调用成员方法

Runtime也不能直接使用new来创建对象，要使用静态函数getRuntime()来获取.

==Runtime中的方法不是静态的，但是getRuntime是静态的。==

==也就是说，getRuntime方法可以这么调用：Runtime.getRuntime==

==这种写法就可以不用创建Runtime对象，直接调用Runtime的方法==

==例如：Runtime.getRuntime.exit(0)==

Object和Objects

Object

在java中所有的类都直接或者间接继承Object类

也就是说想使用Object类的成员属性和方法，直接调用就行.可以被所有子类访问

==当equals方法的调用对象是null时，会报错，所以自己要用if判断调用equals的对象是否为null==

==这样很麻烦==

==所以可以使用Objects中的equals方法==

clone方法

clone方法是Object的成员方法

但是clone方法是provide(受保护的)，只能同一个包中的子类和父类可以访问.

也就是说，不能直接在我们自己写的程序中通过Object对象直接调用clone方法

要使用就必须在自己的类中重写clone方法

包装类

如何理解包装类？

包装类就是对基本数据类型进行加工，写好一些常用的功能.

从内存的角度看，基本数据类型是直接在栈里开空间.包装类是堆里new一个空间，再把基本数据类型存进去.

我们知道在集合的使用中，基本数据类型是不能直接放进集合中的，而包装类可以.

包装类的数据是引用数据类型

Integer

==不管在jdk5以前还是以后，在Integer的底层会提前创建好一些整形（char的取值范围的整形）==

==当new到这些整形的时候，就直接调用这些整形，不需要重新开内存==

JDK5以前

有二种方式创建Integer对象

1.利用构造方法创建

Integer a = new Integer(123);

2.利用静态方法创建

Integer a = Integer.valueOf(123);

包装类如何进行计算

1.把数据类型进行拆箱（把包装类变成基本数据类型）

2.计算

3.将计算完的结果装箱（把基本数据类型变回包装类）
