拆装箱

在集合中，基本数据类型是不能作为数据类型直接用，要变成对应的包装类.

JDK5以前

jdk5以前使用基本数据类型作为集合的数据类型时，需要手动将元素先变成包装类然后在添加到集合中

//手动装箱
//将基本数据类型变成包装类
Integer i1 = Integer.valueOf(1);
Integer i2 = Integer.valueOf(2);
Integer i3 = Integer.valueOf(3);
Integer i4 = new Integer(4);
Integer i5 = new Integer(5);
//想要包装类变回基本数据类型，要进行手动拆箱
int i = i1.intValue();
//创建集合
ArrayList<Integer> a = new ArrayList<>();
//添加元素
a.add(i1);
a.add(i2);
a.add(i3);
a.add(i4);
a.add(i5);

JDK5以后

java为了方便做了优化，直接用包装类就可以.

底层还是拆箱装箱，只不过java自动帮你完成，不需要在自己手动拆装箱.

ArrayList的五种遍历方式

以自定义类型举例.

import java.util.*;
import java.util.function.Consumer;

class Student {
    String name;
    int age;
    Student(int age,String name) {
        this.age = age;
        this.name = name;
    }
    //重写equals
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Student student = (Student) o;
        return age == student.age && Objects.equals(name, student.name);
    }
    //重写toString
    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}
public class Test09 {
    public static void main(String[] args) {
        //创建集合
        ArrayList<Student> al = new ArrayList<>();
        //创建对象
        Student s1 = new Student(10,"张三");
        Student s2 = new Student(20,"李四");
        Student s3 = new Student(30,"王五");
        Student s4 = new Student(10,"张三");
        //给集合添加元素
        al.add(s1);
        al.add(s2);
        al.add(s3);
        //1.普通for遍历
        for(int i = 0;i<al.size();i++) {
            System.out.print(al.get(i)+" ");
        }
        System.out.println();
        //2.增强for遍历
        //增强for中的变量（student）是集合中数据的拷贝
        //所以对增强for中的变量进行修改不会影响集合中原本数据.
        for (Student student : al) {
            System.out.print(student+" ");
        }
        System.out.println();
        //3.lambda表达式遍历（forEach）
        /*al.forEach(new Consumer<Student>() {
            public void accept(Student s) {
                System.out.print(s+" ");
            }
        });
        System.out.println();*/
        //lambda简化
        //底层就是java写好的for循环遍历集合.
        al.forEach( s-> {System.out.print(s+" ");});
        System.out.println();
        //4.迭代器
        //如果想在使用了迭代器的while中对集合进行删除等操作，需要使用迭代器的方法，而不是集合中的方法
        Iterator<Student> it = al.iterator();
        //hasNext判断还有没有元素.
        while(it.hasNext()) {
            //next功能是获取当前元素，然后指向下一个元素
            System.out.print(it.next()+" ");
            //用迭代器，而不是集合对象
            //it.remove();//正确使用，删除的是当前迭代器指向的元素
            //al.remove(s1);//错误使用
        }
        System.out.println();
        //5.列表迭代器ListIterator
        //和普通迭代器比，多了几个方法.使用方法和迭代器一样
        //列表迭代器多了个添加方法，迭代器在遍历的时候只能删不能加.
        //列表迭代器可以从后往前访问元素，迭代器只能从前往后.
        ListIterator<Student> lIt = al.listIterator();
        while(lIt.hasNext()) {
            System.out.print(lIt.next()+" ");
        }
    }
}

Set

Set集合的特点：添加的元素无序（指存和取的顺序）、不重复、无索引

HashSet集合

元素存取的顺序无序，元素不能重复，没有索引

HashSet底层使用哈希表存储数据

哈希表是一种对于数据增删改查性能较好的数据结构

哈希表的组成

JDK8以前是：数组+链表

JDK8以后是：数组+链表+红黑树

成员方法

底层原理

JDK8以后，当链表的长度超过8，且数组长度大于等于64时，自动转换成红黑树

如果集合中定义的是自定义对象，必须要重写hashCode和equals方法

HashSet的三个问题

为什么HashSet存和取的顺序是无序的？

因为当有同样的数据存入是，会挂在老数据的下面，而读取时读取的顺序是固定的，所以是无序的

HashSet为什么没有索引？

因为HashSet的底层不只是数组，而是数组+链表，数组的同一个索引上可能有很长的一串链表.

当通过索引获取元素是，你不能确定获取的元素是不是你要的.

所以干脆取消的索引.

HashSet是利用什么机制保证数据去重的？

使用HashCode方法，该方法计算出哈希值来确定元素应该添加在数组的哪个位置，

然后在通过equals方法比较对象内部的属性值是否相同.

LinkedHashSet集合

有序（存入和取出的顺序），不重复，无索引

底层原理

基本组成原理和HashSet相同

不同的是：每个元素额外多了一个双链表机制记录存储的顺序.

总结：

需要保证数据唯一且无序用HashSet

需要保证数据唯一缺=且有序用LinkedHashSet

自定义类型

重写排序方法，实现Comparable

class Student implements Comparable<Student>{
    String name;
    int age;
    Student(int age,String name) {
        this.age = age;
        this.name = name;
    }
    //重写equals
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Student student = (Student) o;
        return age == student.age && Objects.equals(name, student.name);
    }
    //重写toString
    @Override
    public String toString() {
        return "Student{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
    //重写hashCode
    @Override
    public int hashCode() {
        return Objects.hash(age, name);
    }
    //重写TreeSet排序方法

    @Override
    public int compareTo(Student o) {
        return this.age-o.age;
    }
}
public class Test09 {

        Student s4 = new Student(40,"赵六");

        ts.add(s3);
        ts.add(s4);

拓展

java中的泛型是违泛型.怎么理解？

在.java文件中是有泛型的，但是在编译后的class文件中是没有泛型的.

泛型的原理就是在添加元素时对添加的元素类型进行检查.如果不对就报错，但是实际底层还是用Object保存.

当需要取出元素的时候，java根据泛型中写的类型自动从Object转换成对应的数据类型.

        //在添加数据的时候，如果键存在，那么将原有的覆盖，会把被覆盖的值进行返回
        m.put("11","123");
        m.put("22","234");
        m.put("33","345");
        m.put("44","456");
        //默认打印
        System.out.println(m);
        System.out.println("1.键找值------------------------- --");
        //三种遍历方式：1.键找值、2.键值对、3.lambda表达式
        //键找值
        //将键放入一个Set集合中，用Set集合中的键遍历Map集合
        //Map集合中已经有写好的将键放入Set集合的方法
        Set<String> s1 = m.keySet();
        for (String ss1 : s1) {
            System.out.print(ss1+" ");
            //通过Set中的Key,找到对应的值
            System.out.print(m.get(ss1)+" ");
        }
        System.out.println();
        Iterator<String> it1 = s1.iterator();
        Iterator<String> it2 = s1.iterator();
        while(it1.hasNext()) {
            System.out.print(it2.next()+" ");
            System.out.print(m.get(it1.next())+" ");
        }
        System.out.println();
        s1.forEach(new Consumer<String>() {
            @Override
            public void accept(String s) {
                System.out.print(s+" ");
                System.out.print(m.get(s)+" ");
            }
        });
        System.out.println();
        System.out.println("2.键值对----------------------------");
        //键值对
        //键找值是通过一个方法将键存进Set集合
        //而键值对是用一个方法将Map中的键和值一起变成对象，存进Set中
        Set<Map.Entry<String,String>> s2 = m.entrySet();
        //然后遍历Set集合
        for (Map.Entry<String, String> ss2 : s2) {
            //用自动生成的get方法获取到键值
            System.out.print(ss2.getKey()+" ");
            System.out.print(ss2.getValue()+" ");
        }
        System.out.println();
        s2.forEach(new Consumer<Map.Entry<String, String>>() {
            @Override
            public void accept(Map.Entry<String, String> stringStringEntry) {
                System.out.print(stringStringEntry.getKey());
                System.out.print(stringStringEntry.getValue());
            }
        });
        System.out.println();
        Iterator<Map.Entry<String,String>> it3 = s2.iterator();
        Iterator<Map.Entry<String,String>> it4 = s2.iterator();
        while(it3.hasNext()) {
            System.out.print(it3.next());
        }
        System.out.println();
        System.out.println("3.lambda表达式----------------------------");
        m.forEach(new BiConsumer<String, String>() {
            @Override
            public void accept(String s3, String ss3) {
                System.out.print(s3+ss3);
            }
        });
        m.forEach((s4,ss4)-> System.out.print(s4+ss4));

    }
}

格式：数据类型...变量名

底层是一个数组.

底层就创建了一个E类型的数组e,然后将参数放进数组.

一个方法中只能写一个可变参数

如果一个方法中除了可变参数还有其他参数，那么可变参数要写在最后

不可变集合

不可变集合不能被添加、删除或修改

==java9以后才有的不可变集合==

应用场景

  • 如果某些数据你不想让他在程序运行中发生改变，可以放到不可变集合中

  • 如果集合被不可信任的库调用，不可变集合是安全的

创建格式

List创建不可变集合

List<数据类型> li = List.of(数据,数据,...)

Set创建不可变集合

Set<数据类型> set = Set.of(数据，数据，...)

Map创建不可变集合

数据小于10对

Map<String,String> map = Map.of(键，值，键，值，键，值，...，...)

  • Map集合添加元素必须是键和值成对出现
  • Map不可变集合最多只能传20个键和值（键和值加起来20个，不是20对）
数据大于10对

数据大于10，就要使用另一个成员方法ofEntries()

1.创建一个普通的HashMap集合

  • HashMap<String,String> hm = new HashMap<>()

2.使用put方法添加超过10对的数据

3.将双列集合变成键值对形式的单列集合

  • Set<Map.Entry<String,String>> set = hm.entrySet()

4.把得到的单列集合变成数组（调用成员方法toArray）

  • set.toArray()

因为toArray默认的返回值是Object数组类型，而我要的是Entry对象数组

所以，使用有参的toArray方法，他会根据参数来决定返回值的类型.

也就是说，我传的参数是Entry对象数组类型，那么toArray的返回值也是Entry对象数组类型

最后得的：Map.Entry[] arr = set.toArray(new Map.Entry[0])

5.使用不可变集合的方法将数组变成不可变集合

  • Map m = Map.ofEntries(arr)
