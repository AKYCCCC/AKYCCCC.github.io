java_IO流

在之前File的操作中，仅仅对文件的一些信息进行获取

而想要对文件进行读写，就需要使用IO流

问:什么是IO流

答：存储和读取数据的解决方案

I：input

O : output

流：传输数据

问：IO流的作用

答：用于数据的读写（本地文件或网络）

IO流的划分

按照流向分分为：1.输出流，2.输入流

1.输出流：程序向文件输出内容（output）

2.输入流：文件向程序输入内容（input）

按照操作文件类型分为1.字节流，2.字符流

1.字节流：可以操作所有类型的文件

2.字符流：只可以操作纯文本文件（.txt、.md、.xml、.lrc等）

IO流操作步骤

1.创建对象

2.操作数据写入或读取

3.释放资源（关闭流）

创建对象

FileOutputStream fos = new FileOutputStream();

FileInputStream fis = new FileInputStream();

  • 可以直接输入字符串表示路径
  • 可以输入File对象表示路径

写出数据

write();

  • write中的参数是整形，写入时转换成ASCLL码值对应的字符

read()

  • 每次只读取单个字符，返回字符ASCLL码值

释放资源

fos.close();

  • 使用完就关闭流，不然文件会是运行中状态

输出Output

1.创建对象

FileOutputStream()

  • 记得抛出异常
  • 可以直接输入字符串表示路径
  • 可以输入File对象表示路径
  • 保证父级路径正确的情况下，如果没有该文件，就会自动创建一个文件
  • 如果父级路径不正确直接报错
  • 如果文件已经存在，则会清空文件中原本的内容.

2.写出数据

write(int a)

  • 一次写一个字符

write(byte[] arr)

  • 一次写一个字节数组字符

write(byte[] arr，int off,int len)

  • 一次写一个字节数组的部分数据

  • 参数一：字节数组，参数二：起始位置，参数三：表示数据个数

换行写

换行写本质就是往文件里写一个回车一个换行

回车：\r

换行：\n

换行的本质是两步，第一步使用回车将光标移到开头，第二步换行.所以是”\r\n”

续写

如果想要续写，打开续写开关就行

续写开关在创建对象的第二个参数默认是false

FileOutputStream fos = new FileOutputStream(文件路径，续写开关)

3.关闭资源

close()

输入Input

1.创建对象

FileInputStream()

  • 记得抛出异常
  • 可以直接输入字符串表示路径
  • 可以输入File对象表示路径
  • 如果文件不存在，或者路径错误，直接报错

2.读取数据

read()

  • 每次只读取一个字符，返回字符的ASCLL码值
  • 读不到返回-1

read(byte[] arr)

  • 一次读取一个字节数组数据

  • 读取的时候会尽可能的将数组装满

  • 数组的长度自己控制（推荐1024的整数倍）

  • 返回读取到的字节数

  • 读取不到返回-1

read(byte[] arr,int off,int len)

循环读取

3.释放资源

close()

  • 使用完记得关闭

拷贝文件

循环拷贝

底层就是用循环，一边读一边写.

字节数组拷贝

字节数组读取注意点

读取内容的大小比字节数组的容量大

如果内容比数组大，那么可能会需要读取2次或多次

在读取最后一次数据的时候，如果数据的数量不是刚好和数组的容量一样大，那么数据就会出现错误的写入

例：Test03.txt中的内容是26个英文字母.使用20个字节的数组将内容拷贝到copy.txt中

造成这样的结果就是，每次读取数据是覆盖掉数组中的内容，而不是一个新的空的数组获取.

因为数组大小是20，数据有26个，第二次只覆盖了前6个，所以其实第二次读取完后数组中的内容是uvwxyzghijklmnopqrst

写入的时候也就会造成错误的原因

解决办法，通过read的返回值，规定写入的数量就好了
