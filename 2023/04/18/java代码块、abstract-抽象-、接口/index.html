java代码块、abstract(抽象)、接口

java学习
java
发布日期:   2023-04-18

代码块

局部代码块

问：什么是局部代码块？

答：被大括号{}包起来的就是局部代码块

局部代码块定义

可以写在主函数中

可以写在成员方法中

可以写在类中

//局部代码块
{
    int a;
    System.out.println(a);
}

局部代码块的特性

局部代码块只会在调用时被调用一次，调用完就回收.无法被二次使用

局部代码块中定义的变量，只能在他自己的范围内使用

外部定义的方法和变量可以在局部代码块中使用

局部代码块的执行顺序

局部代码块在主函数中，按照主函数的执行顺序，从上至下.

局部代码块写在类中，按照类的执行顺序，从上至下.创建一个类就会执行一遍局部代码块

局部代码块写在构造方法中，优先执行类中完类中全部局部代码块，不管类中的代码块的位置先后，最后执行构造方法中的代码块

局部代码块写在成员方法中，只有调用时才会执行，调用一次方法，就执行一次.

局部代码块的好处

用完就回收，节省内存空间.

随着内存的发展，已经不需要这么省内存了.

如果使用局部代码块只是单纯想节省空间，不建议用局部代码块.没必要去节省这么点空间.

构造代码块

当无参构造和有参构造中有同样的代码，就可以将同样的代码抽取出来

然后放在局部代码中.局代码块放在类的成员位置.

构造代码块：就是写在类成员位置的局部代码块，代码块中的内容就是构造方法中同样的代码.

静态代码块

关键词static

语法：static{}

在局部代码块前面加上static

静态代码块的特性

静态代码块随着类的加载而加载，且只执行一次。

无论创建多少个类，只会执行一次.

静态代码块：

  • 不可以写在主函数中

  • 不可以写在构造方法中

  • 不可以写在成员方法中

  • 只能写在类的成员位置

静态代码块执行的顺序

静态代码块写在类中，在创建类的对象时被最先调用.然后在类调用构造方法.且不论创建多少个类，只会调用一次.

静态代码块的实际应用

用于一次性数据的初始化.

abstract(抽象)

子类继承了父类，子类可以重写父类的方法，也可以不重写父类的方法.

如果加上了abstract，那么子类就必须重写父类的方法.

抽像的定义

  • 抽象方法定义

    public abstract 返回值类型 方法名 (参数列表) {......}

  • 抽象类定义

    public abstract 类名 {......}

抽象类和抽象方法的特性

抽象类

  • 抽象类不能被实例化
  • 抽象类中的所有成员方法必须全部被子类重写
  • 抽象类不能被实例化，但是可以有构造方法.

抽象方法

  • 一个类中只要有一个抽象方法，这个类就是抽象类.类上必须加上abstract
  • 一个类是抽象类，类中的方法可以不是抽象.
  • 抽象方法不能有方法体
  • 抽象方法一定要被子类重写

抽象类的意义

统一子类中的方法格式和方法名字.

如果父类中的某些方法一定要在子类中重写，那么可以用抽象方法强制让子类重写.

抽象类最主要是用来被子类继承，然后重写父类中的方法.

接口interface

一个父类，有众多个子类.

子类中又有自己的独特的成员方法.

但是不同的子类会有同样的方法，也会有不同的方法.

那父类中就无法抽取出子类中所有的方法.

例：

动物类：动物一般都会吃饭睡觉，所以动物类可以抽取出吃饭睡觉的方法.

鱼类是动物，鱼可以在水里呼吸

狗类是动物，狗不能在水里呼吸

所以作为父类的动物类，就无法把在水里呼吸给抽取出来.只能在鱼类中单独实现.

但不只鱼会在水中呼吸.如果都在各自类中实现的话，又会导致方法格式的不统一

所以就有了接口

将“在水中呼吸”作为一个方法抽取出来，作为一个接口.

所有能“在水中呼吸”的类都实现”在水中呼吸”的这个接口

接口的定义

public interface 接口名 {......}

接口一般单独写在接口文件中

接口的特性

继承只能继承一个类.

接口可以被无限多个实现.

接口不能被实例化

接口是抽象类

接口成员属性的特性

只能是常量：被public static final修饰.

接口中成员属性默认被public static final修饰，被这么修饰的成员属性，必须一开始就初始化值.

接口成员方法特性

所有成员方法默认都被 public abstract修饰

JDK7以前

接口中的成员方法只能且全都是抽象方法.

JDK8的新特性

接口中可以定义有方法体的方法

接口中可以定义有方法体的静态方法

JDK9的新特性

接口中可以定义有方法体私有的方法.

接口构造方法特性

接口不能有构造方法

接口小技巧：接口适配器（Adapter）

原理，创建一个中间类继承接口，然后重写接口中所有方法.

然后将中间类定义为抽象类，对中间类中实际需要用到的方法定义为抽象方法.

然后再用真正的类去继承这个中间类.

这个中间类就是 接口适配器(Adapter)

例

//接口要单独写在一个文件中.这方便看才写在一起
//创建一个接口
public interface testInterface {
    public void func01();
    public void func02();
    public void func03();
}
//使用接口适配器.实现接口，然后单独将func02方法变成抽象方法
abstract class SonAdapter implements testInterface {
    @Override
    public void func01() {
        //空实现
    }
    
    @Override
    public abstract void func02();//单独变成抽象方法
    
    @Override
    public void func03() {
        //空实现
    }
}
//Son类只想用接口中的func02方法.不想重写01和02方法.
class Son extends SonAdapter{
    //Son继承接口适配器，就只需要重写func02方法
    @Override
    public void func02() {
        System.out.println("666");
    }
}
public class Main {
    public static void main(String[] args) {
        Son s = new Son();
        s.func02();
    }
}

JDK8接口新特性

新特性1：接口中的成员方法可以有方法体

定义格式

加上default修饰

public interface testInterface {
    //修饰以后就可以有方法体了.
    public defaule void func01()
    {
        System.out.println("666");
    }
}

特性

被default修饰的抽象方法，可以不被重写

实际作用

接口升级，如果一个接口中新增了方法.那么所有用到这个接口的类都需要重写接口中新的抽象方法.

加上default以后，实现了这个接口的子类就可以不重写接口中新增的抽象方法.

如果有需要，以后再把default去掉就可以了.

新特性2：接口中可以定义有方法体的静态方法

定义格式

public interface testIterface {
    //静态方法
    public static void func01() {
        System.out.println("666");
    } 
}

接口静态方法调用

接口名.方法名

用接口名点方法名直接调用.

静态方法和子类重新的区分

实现了接口的子类中，有和接口中静态方法重名的成员方法.

注意，这不是重写，只是子类中有一个和接口中静态方法重名的方法而已.不是重写.

JDK9接口新特性

接口中可以定义有方法体的私有方法.

普通私有方法定义

public interface testIterface {
	private void func01() {
    	sout("666")
	}
}

静态私有方法定义

public interface testIterface {
	private static void func01() {
    	sout("666")
	}	
}

实际作用

普通私有方法给接口中的普通方法（default）服务

静态私有方法给接口中的静态方法（static）服务
